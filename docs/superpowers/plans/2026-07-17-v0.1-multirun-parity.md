# mandor v0.1 — Multirun Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A single static Zig binary that spawns N workers from argv, forwards signals, reaps zombies, restarts per policy with exponential backoff, and exits with the worst worker exit code.

**Architecture:** Single-threaded poll loop. All forwardable signals blocked and consumed via `signalfd`; SIGCHLD drives a `waitpid(-1, WNOHANG)` reap drain; restart deadlines computed from `clock_gettime(MONOTONIC)` set the poll timeout. Raw `std.os.linux` syscalls only — no libc, no `std.Io` runtime, zero steady-state allocation (fixed worker table + fixed command buffers).

**Tech Stack:** Zig 0.16.0 (pinned in `.zigversion`), `std.os.linux` syscall layer, `main(init: std.process.Init.Minimal) u8` entry (argv arrives as `[]const [*:0]const u8`, environ as `[:null]const ?[*:0]const u8` — both directly usable by `execve`).

## Verified 0.16 API notes (differences from older Zig)

- `std.posix` no longer has `fork`/`waitpid`/`execvpe`/`pipe2`/`write` — use `std.os.linux.{fork,waitpid,pipe2,execve,write,read,kill,clock_gettime}` which return `usize`; decode errno with `std.os.linux.E.init(rc)`.
- `std.posix` DOES still have: `sigprocmask`, `sigemptyset`/`sigaddset` (return/take `sigset_t`), `signalfd(fd, *const sigset_t, flags) !fd_t`, `poll([]pollfd, timeout) !usize`, `kill`, `read`.
- Wait status: `std.os.linux.W.IFEXITED(s)/EXITSTATUS(s)/IFSIGNALED(s)/TERMSIG(s)`.
- Signals are typed: `std.posix.SIG` is an enum-like struct; `sigaddset(&set, .TERM)` style.
- stdout/stderr writers require an `Io` instance in 0.16 (writergate). v0.1 avoids them: all supervisor logging via `std.debug.print` (stderr, lock-protected, no Io needed).
- `main` may be `pub fn main(init: std.process.Init.Minimal) u8` — no allocator, no Io spun up. Exit code = return value.

## CLI grammar (v0.1)

```
mandor [--restart=never|on-failure|always] [--backoff-max=DUR] [--] "CMD" ["CMD"...]
mandor --help | --version
```

- Each positional arg is one worker command string, tokenized by mandor
  (whitespace split; single/double quotes group; `\"` and `\\` escapes inside
  double quotes; no variable expansion — scratch images have no shell).
- `--restart` default `never`. `--backoff-max` default `30s`. DUR accepts
  `NNNms|NNs|NNm` (integer only).
- Worker name = basename of token[0], truncated to 31 bytes; duplicate names
  get `-2`, `-3` suffix.
- PATH lookup: if token[0] has no `/`, child tries `execve` against each PATH
  dir; on total failure child `_exit(127)` after a stderr message.

## Runtime behavior contract

- Restart decision: `never` → no; `on-failure` → restart iff exit code != 0 or killed by signal; `always` → always; never restart while shutting down.
- Backoff: first restart after 200ms, doubling, capped at `--backoff-max`; reset to 200ms when the previous run lived >= 10s.
- Worker exit status → code: exited(c) → c; signaled(sig) → 128+sig.
- mandor exits when no workers are live and none are pending restart; exit code = max over every worker's final status code (0 if no workers ever failed).
- SIGTERM/SIGINT to mandor: forward same signal to all live workers, enter shutdown (cancel pending restarts). Second TERM/INT: SIGKILL all. Exit when all reaped.
- SIGHUP: forward to workers, no state change.
- Unknown reaped PIDs (PID-1 orphans): reap silently.
- Max 64 workers, max 64 argv tokens per command, 4KB command buffer per worker (fixed arrays; overflow = startup error, exit 2, usage message).

## File structure

- `src/main.zig` — `main(Init.Minimal) u8`; converts argv, calls `cli.parse`, on Linux runs `supervisor.run(cfg)`; non-Linux prints "linux only" and exits 2. Also the root test aggregator (`test { _ = @import(...); }`).
- `src/cli.zig` — pure: `parse([]const []const u8) ParseError!Config`, `parseDuration([]const u8) ?u64` (ms), tokenizer `tokenize(cmd: []const u8, buf: []u8, argv_out: [][]const u8) TokenizeError![]const []const u8`. Unit-tested cross-platform.
- `src/backoff.zig` — pure: `next(prev_delay_ms, uptime_ms, max_ms) u64`, `shouldRestart(policy, status) bool`, `statusToCode(status) u8`. Unit-tested cross-platform.
- `src/signals.zig` — Linux: block {TERM,INT,HUP,CHLD}, create signalfd, `read` → drained `enum {term_int, hup, chld}` events.
- `src/spawner.zig` — Linux: `Worker` struct + fixed table; `spawn(*Worker)` fork/exec with PATH candidates; child unblocks signals before exec.
- `src/reaper.zig` — Linux: `drain(table) ReapSummary` — waitpid WNOHANG loop, classify statuses into worker table.
- `src/supervisor.zig` — Linux: poll loop, restart scheduling, forwarding, exit-code computation. (CLAUDE.md layout lists main/spawner/reaper/signals; supervisor.zig is the glue loop kept out of main.zig so main stays portable.)
- `test/harness/` — `crash_exit.sh`, `stable.sh`, `flappy.sh` + `run_tests.sh` integration assertions (run in WSL; CI containers later).

## Interfaces (exact)

```zig
// cli.zig
pub const RestartPolicy = enum { never, on_failure, always };
pub const Config = struct {
    restart: RestartPolicy = .never,
    backoff_max_ms: u64 = 30_000,
    commands: []const []const u8,   // slices into original argv strings
    help: bool = false,
    version: bool = false,
};
pub const ParseError = error{ UnknownFlag, BadValue, NoCommands, TooManyWorkers };
pub fn parse(args: []const []const u8, cmd_storage: *[64][]const u8) ParseError!Config;
pub fn parseDuration(s: []const u8) ?u64; // ms; "500ms"|"30s"|"2m"
pub const TokenizeError = error{ Empty, UnterminatedQuote, TooManyArgs, TooLong };
pub fn tokenize(cmd: []const u8, buf: []u8, argv_out: [][]const u8) TokenizeError![]const []const u8;

// backoff.zig
pub fn next(prev_delay_ms: u64, uptime_ms: u64, max_ms: u64) u64; // 0 prev → 200
pub fn shouldRestart(policy: cli.RestartPolicy, exited_clean: bool) bool;

// reaper.zig (linux)
pub const Status = union(enum) { running, exited: u8, signaled: u8 };

// spawner.zig (linux)
pub const Worker = struct {
    name: [32]u8, name_len: u8,
    cmd_buf: [4096]u8, argv: [65]?[*:0]const u8, argc: u8,
    pid: i32, // 0 = not running
    status: reaper.Status,
    restarts: u32,
    last_start_ms: u64, next_restart_ms: u64, cur_delay_ms: u64,
    final_code: u8,
};
```
