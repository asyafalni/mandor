# mandor v0.2 — Capture + Stats Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. Checkbox steps.

**Goal:** Per-worker stdout/stderr capture through fixed ring buffers with `[name]`-prefixed passthrough, a /proc sampler (CPU%, RSS, fds, threads), and `mandor report` (human + `--json`) fed by a state file the supervisor maintains.

**Architecture:** Workers get stdout/stderr pipes (`pipe2` NONBLOCK|CLOEXEC, child `dup2`s). The existing poll loop grows to `1 + 2×workers` fds; readable pipe data is line-assembled into per-worker fixed accumulators, echoed to mandor's stdout as `[name] line`, and framed into a per-worker 256 KB byte ring. Every 5 s the sampler parses `/proc/<pid>/stat` (+ fd dir count) into a per-worker rolling window of 24 samples. Supervisor serializes worker state + stats to `<state-dir>/state.json` (tmp+rename) on each sampler tick and worker death; `mandor report` just reads and formats that file. Everything stays single-threaded, allocation-free, raw syscalls.

**Global constraints:** same as v0.1 plan (Zig 0.16.0 pinned, <500 KB stripped, no panics on supervision path, zero steady-state alloc, no deps, plain commit messages).

## Decisions locked here

- One combined 256 KB ring per worker (stdout+stderr), line-framed records
  `[u16 len][u8 flags][bytes]`; flags bit0 = stderr, bit1 = error/warn-looking
  (v0.3 fills heuristics; v0.2 sets bit0 only). Oldest records evicted on wrap.
- Max line 4 KB (per-stream accumulator); longer lines are split, remainder
  flagged as continuation (flags bit2).
- CPU% uses USER_HZ=100 (constant on every mainstream Linux arch; no libc
  sysconf available by design).
- State dir: `--state-dir=PATH` flag, default `/var/lib/mandor`, env
  `MANDOR_STATE_DIR` overrides default; unwritable → warn once, continue
  (report then says "no state available").
- `mandor report [--json] [--state-dir=PATH]` is a subcommand: first
  positional arg `report` before flags.
- Echo write is one `write(1, ...)` per line (atomic ≤ PIPE_BUF).

## Files

- Create `src/ring.zig` — pure framed ring buffer + iterator (unit tests).
- Create `src/capture.zig` — Linux: pipe setup helper, per-worker line
  assembler feeding ring + stdout echo (assembler logic pure + tested).
- Create `src/sampler.zig` — `/proc` text parsers pure + tested with fixture
  strings; Linux readers (`/proc/<pid>/stat`, fd count via getdents64).
- Create `src/report.zig` — state JSON serializer (fixed 64 KB buffer,
  hand-rolled, pure + tested) + reader/formatter for the subcommand.
- Modify `src/spawner.zig` — Worker gains pipe fds + ring + accumulators +
  stats window; spawn wires `dup2`.
- Modify `src/supervisor.zig` — poll set, sampler cadence, state writes.
- Modify `src/cli.zig` + `src/main.zig` — `report` subcommand, `--state-dir`.
- Extend `test/harness/run_tests.sh` — prefix assertions, report smoke.
