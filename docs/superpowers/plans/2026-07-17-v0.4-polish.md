# mandor v0.4 — Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** mandor.toml config (CLI-only always works), cgroup v2 OOM cause
detection, optional hand-rolled Prometheus text endpoint, CI with size gate.

**Decisions:**
- Config precedence: TOML < env < CLI. `--config=PATH` explicit; otherwise try
  `./mandor.toml` — no config file is the fully supported default.
- TOML subset (documented in README): `key = "string" | integer`, string
  arrays `workers = ["cmd", ...]` (single or multiline), `#` comments. Keys:
  `restart`, `backoff_max`, `state_dir`, `metrics_port`, `workers`.
- OOM: read `/sys/fs/cgroup/memory.events` `oom_kill` counter; if it rose
  since the last death-check and the worker died by SIGKILL → cause `oom`
  (schema slot already reserved). Pure parser + counter diff, tested.
- Metrics: `--metrics=PORT` (default off) binds 127.0.0.1:PORT, joins the
  poll loop, one route: any HTTP request → 200 text/plain with
  `mandor_worker_up/restarts_total/rss_kilobytes/cpu_percent/fds/threads` and
  `mandor_incidents_total`. One connection at a time, no keep-alive.
  (CLAUDE.md explicitly allows this one optional local endpoint.)
- CI: GitHub Actions — zig build test, harness, musl x64+arm64 ReleaseSafe,
  stripped-size gate < 500 KB.

**Tasks:**
1. config.zig TOML subset (TDD) + wiring into main (precedence) → commit.
2. cgroup.zig oom_kill parser + supervisor cause override (TDD parser) → commit.
3. metrics.zig socket + text renderer (renderer TDD) + poll-loop wiring +
   harness case (curl the endpoint) → commit.
4. `.github/workflows/ci.yml` + README (config docs, status) + size check +
   merge + push → done.
