# mandor v0.6 — Liveness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** ROADMAP Tier 2: structured trace frames, command health checks,
s6-style readiness, ELF build-id. One schema bump to `"v": 4`.

## Schema v4 changes (single bump, golden-locked)

- `trace.frames`: strings → objects `{"function","file","line","in_app"}`
  (Sentry frame vocabulary — the 15–17× localization lever).
- `process.ready`: bool — worker had signaled readiness before the incident.
- `process.build.elf_build_id`: hex GNU build-id of the resolved exe ("" if
  absent/unparseable).
- `cause.kind` gains `unhealthy` (string enum extension, no shape change).

## Decisions

- **in_app heuristics:** go: function not `runtime.*`; rust: function not
  `core::`/`std::`/`rust_begin_unwind` and file not under `/rustc/`;
  python: file not in `site-packages` / `/usr/lib/python`.
- **Health checks:** `--health="NAME=CMD"` (repeatable; TOML `health = [...]`
  array) maps a check command to the worker named NAME. Every
  `--health-interval` (default 30s) mandor forks/execs the check with
  stdout/stderr → /dev/null, 10s hard timeout (SIGKILL). Exit 0 = healthy;
  3 consecutive failures → incident `cause.kind="unhealthy"` + (if
  `--restart-on-unhealthy`) SIGTERM the worker so the normal death/restart
  path takes over. Check pids are tracked separately; reaper reports
  non-worker pids back to the supervisor for matching.
- **Readiness:** `--ready-fd=N` (global). At spawn each worker gets a pipe;
  child dup2s the write end onto fd N. First byte read → `ready=true`,
  logged. Workers that never write simply stay not-ready (only meaningful
  when the flag is set). "died before ready" surfaces via `process.ready`.
- **build-id:** pure ELF note parser (PT_NOTE walk, NT_GNU_BUILD_ID) over the
  first 4 KB of the resolved exe, read once at spawn (cold path).
