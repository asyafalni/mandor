# mandor v0.7 — Tier 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** start-after ordering, incident history that survives supervisor
restarts (schema v5), release/ghcr publishing. Env-allowlist (ROADMAP #16)
intentionally dropped — built-in redaction + simplicity directive.

## Decisions

- **start-after** (TOML only, `start_after = ["dependent=dependency"]`, one
  dependency per worker, chains fine, cycles = startup error): the dependent
  is NOT spawned initially; it starts once the dependency is up — "up" =
  ready (when `ready_fd` is on) or alive ≥ 1 s (stability window otherwise).
  If the dependency dies permanently first, dependents start anyway (warn) —
  no deadlocks. Gating applies to initial start only; restarts are normal.
- **History (schema v5):** `<state-dir>/history.json` — up to 64 entries
  `{sig, first_seen, count, last_seen}` keyed by the dedup signature hash,
  loaded at boot, rewritten atomically on each incident. Bundles gain
  `"history":{"signature":"<hex>","first_seen":"<iso>","count":N}` so the
  LLM can tell "5th time this exact crash since Tuesday" from "brand new".
  In-run cooldown suppression stays in-memory.
- **Release pipeline:** `Dockerfile` (scratch + /mandor), `release.yml` on
  `v*` tags: musl x64+arm64 builds, GitHub Release artifacts, ghcr.io
  multi-arch image `ghcr.io/asyafalni/mandor`.
